Design philosophy

This is a document explaining a few of the major design considerations for Libre Radio. They should, as far is possible, be applied to components of the software suite.

Simple architecture

Each part of the Libre Radio suite should do an easily defined task, so that it without too much trouble can be replaced with a custom built program.

Encourage cooperation

The users of the software suite should, where it is possible, be encouraged to help other users. A practical example of this is integration with the MusicBrainz database in order to simplify the process of cataloguing recordings.
